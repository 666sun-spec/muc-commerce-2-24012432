## 拓展A：导出当前筛选结果
### 一、功能介绍
在数据看板页面增加导出功能，可将下拉框筛选后的品类数据导出为CSV文件，导出内容与页面表格展示数据完全一致；导出文件名携带对应筛选品类信息，同时对下载接口增加登录权限校验，未登录用户无法调用导出功能。

### 二、接口访问方式
1. 页面按钮使用（推荐）
先使用账号`student`、密码`day07`登录系统，进入电商用户数据看板；
在页面顶部下拉框选择需要筛选的品类，页面表格会自动刷新对应数据；
点击右上角【导出当前品类筛选数据】按钮，浏览器会直接下载CSV文件。

2. 浏览器地址直接访问接口
路由地址：`/download`
请求传参格式：`/download?category=品类名称`
示例：
- 导出Fashion品类筛选数据：`http://127.0.0.1:5000/download?category=Fashion`
- 导出全部品类数据：`http://127.0.0.1:5000/download?category=全部`
未登录直接访问该地址会自动重定向至登录页面。

### 三、导出文件规则
1. 数据一致性：导出CSV的字段、数据条目和页面当前筛选表格严格对应；
2. 文件命名：选择「全部」时文件名为`全部_用户品类分析表.csv`，选择单个品类时文件名为`对应品类_用户品类分析表.csv`；
3. 编码格式：CSV文件使用`utf-8-sig`编码，使用Excel打开不会出现中文乱码；
4. 容错处理：若筛选后无匹配数据，接口返回文字提示，不会生成空白CSV文件。

### 四、项目修改内容
1. 后端`app.py`
新增`/download`下载路由函数`download_csv`；
补充导入`pandas`、`io`、`send_file`依赖库；
路由添加`@login_required`登录鉴权装饰器；
复用项目原有`load_dashboard_data`函数复用筛选逻辑，保证前后端筛选条件统一。

2. 前端`templates/dashboard.html`
在页面右上角按钮区域新增导出按钮，绑定`url_for`路由并携带当前选中品类参数。
